# WebFS

> Консольный файловый менеджер для веб-сайтов

---

# Описание проекта

**WebFS (Web File System)** — это консольное приложение на C#, представляющее веб-сайт в виде виртуальной файловой системы.

После подключения к сайту пользователь взаимодействует с ним так же, как с каталогами в Linux. Вместо открытия страниц в браузере приложение анализирует структуру сайта, строит граф ссылок и предоставляет терминальный интерфейс для навигации и исследования.

Главная идея проекта — представить веб-сайт как дерево каталогов, позволяя исследовать его при помощи привычных консольных команд.

---

# Цели проекта

Проект предназначен для изучения современных возможностей платформы .NET через разработку реального CLI-приложения.

В процессе разработки предполагается изучить:

- HttpClient
- Асинхронное программирование (`async/await`)
- `CancellationToken`
- Парсинг HTML
- Построение графов
- Алгоритмы DFS/BFS
- Кэширование данных
- Обработку сетевых ошибок
- Архитектуру CLI-приложений
- Работа с JSON
- Логирование

# Концепция

После запуска пользователь попадает в терминал.

```text
WebFS v1.0

webfs>
```

Подключение к сайту

```text
webfs> open https://example.com
```

После подключения приглашение изменяется

```text
example.com>
```

Теперь пользователь может исследовать сайт, используя привычные Linux-команды.

---

# Основные команды

## Подключение

```bash
open <url>
```

Подключается к сайту и создает новую рабочую сессию.

---

## Навигация

```bash
pwd
```

Показать текущую страницу.

```bash
cd <path>
```

Перейти по ссылке.

```bash
back
```

Вернуться назад.

---

## Просмотр содержимого

```bash
ls
```

Показывает найденные внутренние ссылки текущей страницы.

Пример

```text
/
├── about
├── blog
├── products
└── contacts
```

---

## Построение дерева сайта

```bash
tree
```

или

```bash
tree --depth 2
```

Пример

```text
/
├── Blog
│   ├── 2025
│   └── Archive
├── Products
│   ├── Phones
│   └── Laptops
└── Contacts
```

---

## Поиск

Поиск страницы

```bash
find blog
```

Поиск по шаблону

```bash
find *login*
```

Поиск текста

```bash
grep "CancellationToken"
```

- HTTP Status

---

# Архитектурные принципы

- Каждая команда реализуется отдельным классом.
- Вся бизнес-логика находится в сервисах.
- Команды не работают напрямую с HTTP.
- Все длительные операции являются асинхронными.
- Все длительные операции поддерживают отмену через `CancellationToken`.
- Все данные сайта представлены объектной моделью.
